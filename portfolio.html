// Financial Services · Web Design + Accessibility

Novak Financial Group

Project Screenshot
Replace with Midjourney / real screenshot

// The challenge

The site looked like it was built in 2014. Broken on mobile. No WCAG compliance — a liability for a regulated industry. Referrals were strong but the site was losing warm leads the moment people clicked through.

// The approach

Discovery session to define the audience (high-net-worth individuals, ages 45–65). Mobile-first redesign with clear service lane architecture so visitors could immediately self-segment. Full WCAG 2.1 AA audit and remediation — critical for a financial services client with compliance exposure.

Outcome: 40% reduction in bounce rate in the first 30 days. Zero accessibility violations on launch. Client reported their first website-sourced lead within two weeks.

// Tech stack

HTML/CSS/JS Squarespace CMS Google Analytics WCAG 2.1 AA

// Event Styling · Web Design + Booking

Poppin' by Rosie

Project Screenshot

// The challenge

Rosie had a beautiful Instagram and zero web presence. She was booking everything through DMs, losing track of inquiries, and undercharging because clients couldn't see her full portfolio or pricing upfront.

// The approach

Visual-forward site that let her portfolio do the talking. Custom gallery with accessible image descriptions for every installation. Online inquiry form that captured event type, date, and budget upfront. Clear pricing tiers so she stopped fielding lowball requests.

Outcome: Rosie booked her first corporate event — a $3,200 contract — within 45 days of launch. She credited the pricing page for attracting the right client tier.

// Restaurant · Web Rebuild + Accessibility Remediation

Carmen's Table

Project Screenshot
Replace with Midjourney / real screenshot

// The challenge

PDF menu (a WCAG failure and a mobile nightmare). Site took 8 seconds to load. Hours and location were buried. Carmen was losing walk-in traffic to competitors with better Google presence.

// The approach

Rebuilt the menu as accessible HTML — filterable by dietary restriction, readable by screen readers, fast on mobile. Fixed Core Web Vitals. Structured data markup for Google rich results (hours, menu, reservations). Simple CMS so staff can update specials without a developer.

Outcome: Google ranking for "[neighborhood] restaurant" moved from page 3 to page 1 within 60 days. Load time dropped from 8s to 1.4s. Carmen now updates her own menu.

// Tech stack

HTML/CSS/JS Squarespace CMS Google Structured Data WCAG 2.1 AA
